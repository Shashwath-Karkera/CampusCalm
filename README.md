# CampusCalm

Student Stress Level Prediction Using Machine Learning

CampusCalm is a complete mini-project that predicts student stress level (Low/Medium/High) from academic, personal, and lifestyle factors.

## Stack

- Frontend: Next.js App Router, TypeScript, Tailwind CSS, Framer Motion, Lucide, Recharts
- Backend API: Next.js route handlers
- ML Backend: FastAPI + scikit-learn + pandas + numpy + seaborn + matplotlib + joblib
- Database: NeonDB PostgreSQL via Prisma

## Features

- Premium landing page with modern glass UI
- User registration/login with hashed passwords and JWT cookie sessions
- Student dashboard
- Stress prediction form + risk result + recommendations
- Admin panel + dataset upload
- Analytics dashboard
- Model comparison section
- Prediction history
- PDF report export
- Mood tracker endpoint
- Wellness notifications endpoint
- AI chatbot helper endpoint
- Dark/light theme
- Mobile responsive layout

## Project Structure

- src/app: Next.js routes and pages
- src/components: UI, forms, charts, dashboard modules
- src/lib: auth, prisma, validators, utils, rate limiter, ML client
- prisma/schema.prisma: NeonDB schema
- sql/001_init_schema.sql: migration SQL
- python-backend/train.py: data cleaning, EDA, training, evaluation, clustering, ANN
- python-backend/main.py: FastAPI prediction service
- python-backend/predict.py: model inference
- python-backend/preprocess.py: preprocessing pipeline

## Dataset

- Uses realistic student stress dataset CSV at python-backend/data/student_stress_dataset.csv
- If missing, synthetic data is auto-generated by train.py

## Database Setup (Neon)

1. Create Neon project + database.
2. Copy connection string into .env as DATABASE_URL.
3. Run:

```bash
npm install
npx prisma generate
npx prisma db push
```

Alternatively apply SQL manually from sql/001_init_schema.sql.

## Local Development

1. Frontend/API:

```bash
npm install
npm run dev
```

2. Python ML service:

```bash
python -m venv .venv
.venv\\Scripts\\activate
pip install -r python-backend/requirements.txt
python python-backend/train.py
python python-backend/main.py
```

3. Required environment variables (.env):

- DATABASE_URL
- JWT_SECRET
- PYTHON_ML_URL
- NEXT_PUBLIC_APP_URL

## Deployment

### Frontend + API (Vercel)

1. Push repository to GitHub.
2. Import in Vercel.
3. Add environment variables from .env.
4. Build command: npm run build
5. Output: .next

### Python ML Backend (Railway/Render)

- Start command: uvicorn main:app --host 0.0.0.0 --port 8001
- Root dir: python-backend
- Ensure model file exists by running train.py once during build/deploy

## Reflection and Insights

- Random Forest performs best due to robust handling of mixed feature distributions.
- Sleep, exam pressure, social support, and backlog count are dominant factors.
- Combined academic + lifestyle features significantly outperform academic-only baselines.
- KMeans clusters reveal intervention strategy groups:
	- Academically stressed
	- Lifestyle stressed
	- Balanced
